## 📊 系统架构

### 配图流程

```
1. 文章生成完成
   ↓
2. 分析文章结构（提取##标题）
   ↓
3. 为前 3 个章节生成配图
   ↓
4. 在标题后插入图片 Markdown
   ↓
5. 保存更新后的文章
```

## 📁 工作流程

### 步骤 1：分析文章结构

**逻辑：**
```python
# 使用正则表达式提取##标题
pattern = r'^##\s+(.+)$'
matches = re.finditer(pattern, md_content, re.MULTILINE)

    ...
]
```

### 步骤 2：生成配图

**函数：** `generate_section_image(topic, section_title, style)`

**逻辑：**
1. 构建提示词（见上方模板）
2. 调用豆包 API 生图

**API 调用：**
```python

    base_url="https://ark.cn-beijing.volces.com/api/v3",
    api_key=config['doubao_api_key'],
)

imagesResponse = client.images.generate(
    model="doubao-seedream-4-5-251128",
    prompt=prompt,
    size="2K",
    response_format="url",
    extra_body={"watermark": True},
)
```

### 步骤 3：插入图片

**函数：** `add_images_to_article(article_path, topic, style)`

**逻辑：**
```python
# 为前 3 个章节配图
for section in sections[:3]:
    image_path = generate_section_image(topic, section['title'], style)
    
    # 在标题后插入图片 Markdown
    image_markdown = f"\n\n![{section['title']}]({image_path})\n\n"
    insert_pos = section['position'] + len(f"## {section['title']}")
    content = content[:insert_pos] + image_markdown + content[insert_pos:]
```

**最终效果：**

## 🎨 提示词设计原则

**设计思路：**
- 领域限定 → 避免风格偏离
- 主题 + 章节 → 既统一又有区分
- 色调指定 → 确保专业感

## 📝 最佳实践

### 提示词优化

**✅ 好的做法：**
- 具体明确（包含主题、内容、风格）
- 领域限定（健康科普、科技、教育等）
- 色调指定（温暖、明亮、专业）
- 尺寸要求（符合发布平台）

**❌ 避免的做法：**
- 过于抽象（"画一张好看的图"）
- 缺少上下文（只给标题）
- 风格混乱（多种风格混合）
- 尺寸不符（不符合平台要求）

### 配图数量

**推荐：**
- 短文章（<2000 字）：1-2 张配图
- 中文章（2000-4000 字）：3-4 张配图
- 长文章（>4000 字）：5-6 张配图

**当前策略：**
- 封面图：1 张（必需）
- 章节配图：前 3 个## 标题（可选）
- 分享封面：1 张（可选）
